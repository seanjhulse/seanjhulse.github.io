What is HTTP?

HTTP stands for HyperText Transfer Protocol. If we are willing to get into the weeds a little bit, "hypertext" means "text displayed on a screen", which makes HTTP a lot more logical, right? It's the protocol for transferring text from screen A to a screen B. From a web developer's standpoint, HTTP is only 4 functions: GET, POST, PUT, and DELETE. We don't really care how it happens. If you're really interested, just read Wikipedia and the subsequent details on the official RFC documentation. 99% of web development is just making 4 requests.

We want to GET things, we want to POST (create) things, we want to PUT (update) things, or we want to DELETE things.

If we are going to be honest, web development is literally just doing those 4 things for things tied to a relational database. Sure, some complications come from security and complex relationships in the databse. However, browsers, HTTP, and SQL take 99% of the weight off our shoulders.
